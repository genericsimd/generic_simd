Developer Guide

Get the source code

Make sure you have the access right to yktgsa projects/g/generic_simd directory. Please contact Peng Wu (pengwu@us.ibm.com) if you meet any problems.

Clone in a server that has direct gsa directory access 

 {.sh}
git clone /gsa/yktgsa/projects/g/generic_simd/gsimd.git gsimd

Clone through a server that has gsa directory access. 

 {.sh}
git clone userName@serverName:/gsa/yktgsa/projects/g/generic_simd/gsimd.git gsimd

Source code structure

  • docs Generate html docs through doxygen
  • examples Examples using this library.
  • include The library source code
  • tests Unit tests, test library implementation

Run unit tests

The unit tests are in tests directory. The current unit test covers power_vsx4.h, generic4.h and sse4.h

Please download googletest framework first from https://code.google.com/p/googletest/, and unzip it into "tests/gtest-1.6.0" dirctory. Or you can unzip it to where you want, and modify the "GTEST_DIR" value in tests/Makefile.

Then you can run the test

 {.sh}
$ cd tests
$ make test

The test app will test vsx4, generic4 and sse4 interfaces, and generate the report.

Generate document

Make sure you have doxygen installed. 

 {.sh}
$ cd docs
$ make
$ make publish #You must run publish in a server that has direct gsa directory access.

Working with ISPC

In order to use the library with ispc, please checkout gsimd project in the same directory as ipsc. 

 {.sh}
  .\gsimd
  .\ispc

Then, you just goto ispc\, and execute "TARGET=adapter4 make bench" to compile and run all benchmarks.

Or goto one example, e.g. noise, and execute either "TARGET=adapter4 make" or "make adapter4" to get it compiled.

The current adapter4 is connected to power_vsx4.h
